Basic:

    git config --global user.name Artem
    git config --global user.email artemijklymenko@gmail.com
    git init - creates a new Git repository.
    git status - displays the state of the working directory and the staging area.
    git add - adds a change in the working directory to the staging area.
    git commit - save your changes to the local repository.
    git log - shows a running record of commits.
    git checkout - navigate between the branches created by git branch.
    git branch - create, list, rename and delete branches.
    git push - command is used to upload local repository content to a remote repository.
    git pull - update the local version of a repository from a remote.
    git merge - take the independent lines of development created by git branch and integrate them into a single branch.
    git clone - target an existing repository and create a clone, or copy of the target repository.
    .gitignore - tells git which files (or patterns) it should ignore.

Advanced:

    git revert - forward-moving undo operation that offers a safe method of undoing changes. Instead of deleting or orphaning commits in the commit history, a revert will create a new commit that inverses the changes specified.
    git reset - used to undo local changes to the state of a Git repo/undo changes in your working directory that haven't been comitted yet.
    git rebase - (another way of "merge")integrate changes from one branch onto another.
    git cherry-pick - choose a commit from one branch and apply it onto another.
    git stash - temporarily store your uncommitted local changes - and leave you with a clean working copy, to work on something else and come back later.
